It launches processes, watches live execution, manages approvals, stores reusable configurations, uploads customer-specific sensitive JSON, and attaches approved customer profiles to specific operations so the worker can only unlock the right data for the right run.

Execution and Customer Profile Permissioning
1

Store Customer Profile

The operator uploads customer JSON in the Sensitive Data tab. Backend encrypts the payload, stores it in sensitive_records, and returns copyable vault://profile/field references.

→
2

Attach to This Operation

During launch or corrective prompting, the operator checks the specific customer profiles this run may use. The dashboard sends those selections as sensitiveDataGrants in runtime metadata.

→
3

Persist Operation Grants

The backend merges grants into agent_jobs.metadata. This binds customer-profile permission to the job itself instead of globally unlocking vault access.

4

Worker Uses Vault Ref

The Python worker sees attached profiles in the user message context. When the browser needs PII for a form field, the executor resolves the vault://... reference at runtime.

→
5

Approval + Grant Check

The backend verifies that the requested record is attached to the current job, that requested fields are allowed, and that the operator approves the reveal when step-up is required.

→
6

Reveal Only the Needed Field

Only the approved field value is decrypted for that execution step. Logs stay redacted, and ungranted customer references remain blocked even if a prompt mentions them.

  • Operator sends context or correction
  • Optional new profile grants attached
  • Backend appends metadata + steps
  • Worker resumes with updated state
Why This Dashboard Matters

Centralized Ops

Operators can launch, observe, pause, resume, cancel, and correct many worker-backed jobs from one place.

Human Trust Layer

The dashboard is the live approval and exception-handling surface for risky browser actions and sensitive data access.

Customer Isolation

Each run only receives the customer profiles explicitly attached to it, which is critical when replacing high-volume RPA bots.

Auditable Execution

Replay, screenshots, goal state, operator prompts, and grants all stay tied to the operation record for review and recovery.
